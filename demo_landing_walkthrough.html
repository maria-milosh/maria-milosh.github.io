Harmonica Two‑Phase Pipeline — From Session Creation to Phase‑2 Extractions
This walkthrough explains the full pipeline used in this repo, starting at session creation and ending with extraction output for Phase 2 JSON transcripts. Replace the placeholders with screenshots from your own run so the team can see each stage.
High‑level map

What We’re Doing End‑to‑End

The pipeline has two conversational phases. Phase 1 collects initial opinions and produces a structured extraction file. Phase 2 injects those rephrased ideas back into a new session, collects reflections, and then runs extraction again on the Phase 2 transcript JSON.

Phase 1 — Create session → monitor → save transcript JSON → extract votes/reasons.
Phase 2 — Create session with injected rephrases → monitor → save transcript JSON → extract again.
Inputs

Prerequisites & Config

The scripts read optional defaults from example_pilot.yaml. If present, it provides the topic description, expected participants, and extraction model. You’ll need API keys available in your environment when running the session and extraction steps.

HARMONICA_API_KEY — required for session create + monitor.
OPENAI_API_KEY (or EXTRACTION_API_KEY) — required for extraction.
Phase 1

Create Session & Capture Responses

1

Create a Phase‑1 session

Creates a Harmonica session with a topic prefixed by P1 and writes the session ID into data/responses/phase1_current.txt.

npm run phase1:create -- --config example_pilot.yaml
2

Monitor Phase‑1 until N participants finish

Polls the session, tracks completion, and saves the transcript to data/responses/phase1_<session_id>.json once the max is reached.

npm run session:monitor -- --session-id hst_<PHASE1_ID> --phase 1 --interval 60 --max-users 3
Phase 1

Extract Votes + Reasoning

Extraction reads the Phase‑1 transcript JSON and produces a structured file with votes and reasoning. Output lands at data/responses/phase1_<session_id>_extractions.json.

npm run reasoning:extract -- --session-id hst_<PHASE1_ID> --phase 1 --config example_pilot.yaml
Phase 2

Create Session with Injected Rephrases

The Phase‑2 creation script reads the Phase‑1 extraction file and builds a prompt that injects rephrased participant ideas. It creates a new session whose topic is prefixed by P2.

npm run phase2:create -- --source-session hst_<PHASE1_ID> --config example_pilot.yaml
Phase 2

Monitor Phase‑2 and Save Transcript

Monitor Phase 2 just like Phase 1. When completed, it saves data/responses/phase2_<session_id>.json.

npm run session:monitor -- --session-id hst_<PHASE2_ID> --phase 2 --interval 60 --max-users 3
Phase 2

Apply Extraction to Phase‑2 JSON

This is the final step: run extraction again on the Phase‑2 transcript JSON to produce data/responses/phase2_<session_id>_extractions.json.

npm run reasoning:extract -- --session-id hst_<PHASE2_ID> --phase 2 --config example_pilot.yaml
